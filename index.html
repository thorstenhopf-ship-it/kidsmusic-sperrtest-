SDK-Test: Überlebt die Wiedergabe das Sperren?

Einmaliger Versuchsaufbau. Er beantwortet die eine Frage, an der hängt, ob sich der Umbau von KidsMusic auf einen Player in der Seite lohnt — bevor jemand am Server etwas ändert.

Diese Seite gehört nicht zur App und liegt bewusst außerhalb des Heimnetzes: Spotify verlangt für die Anmeldung HTTPS, und das hat GitHub Pages von Haus aus. Nach dem Test wird sie abgeschaltet. Sie enthält nur die öffentliche Client ID — kein Passwort, kein Secret.

Läuft auf accounts.spotify.com. Dein Passwort sieht diese Seite nie.

3 · Sperren

Nur die beiden fetten Zeilen zählen. Stehen dort etwa gleiche Werte, lief der Ton durch. Ist „Ton gelaufen" deutlich kleiner, hat iOS die Wiedergabe unterwegs beendet — und die Differenz sagt dir, nach wie vielen Sekunden.

So liest du das Ergebnis

  1. Einmal sperren, dann liegen lassen. Jedes Antippen macht die Seite kurz sichtbar und startet eine neue Messung — die kurzen Zwischenwerte verwässern das Bild.
  2. Mindestens zehn Minuten warten. Eine Minute beweist nichts: iOS beendet Hintergrundwiedergabe gern erst nach einigen Minuten.
  3. Danach entsperren und die beiden fetten Zeilen vergleichen.

Gleiche Werte → die Wiedergabe überlebt das Sperren, der Umbau lohnt sich. „Ton gelaufen" deutlich kleiner → iOS hat abgebrochen, und der Deeplink bleibt die bessere Lösung.
